# ReMIX rating app

Human validation for the ReMIX dataset: a React UI plus a small FastAPI backend
in one container.

## Why Docker, and why CommitScheduler

A Space's filesystem is **ephemeral** — wiped on restart, rebuild, and whenever a
free Space sleeps. Writing ratings to a local file would lose them silently.
`huggingface_hub.CommitScheduler` is built for this: it appends to a local JSONL
and pushes to a Dataset repo on an interval, so ratings end up durable and
versioned next to the dataset.

The wire format is the same JSONL schema the Streamlit app writes, so
`scripts/paper_validation_stats.py` reads these ratings unchanged.

## Setup

1. Create a **Dataset** repo for the ratings, e.g. `Pliploop/remix-human-ratings`.
2. Create a **Space** with the Docker SDK.
3. Space → Settings → Variables and secrets:

   | Name | Kind | Value |
   | --- | --- | --- |
   | `DATASET_REPO` | variable | `Pliploop/remix-human-ratings` |
   | `HF_TOKEN` | **secret** | a write token |
   | `EVERY_MINUTES` | variable (optional) | commit interval, default `5` |

Without `DATASET_REPO` and `HF_TOKEN` the app still runs, but ratings stay on the
ephemeral disk and are lost on restart. `/api/health` reports which mode it is in
— check it before sending raters at it.

## Deploying

The build context is the **repo root** (it builds `website/` too):

```bash
git clone https://huggingface.co/spaces/<user>/<space> hf-space
# copy space/* and website/ into it, or push this repo as a remote
docker build -f space/Dockerfile -t remix-space .   # to test the build locally
```

## Data

Rating tasks come from `website/public/data/validation_tasks.json`, generated by:

```bash
python scripts/export_validation_app_data.py
```

That exports **both** sidecars per dataset: the calibration slice (50 sentinel
attention checks + 50 core_overlap items every rater sees) and the full workload
(~1.1k items). The rubric is imported live from
`jamendo_instruct.demo.validation_rubric`, so this app, the Streamlit app and the
LLM judge cannot drift apart.

## Pulling ratings back

```bash
python scripts/sync_human_ratings.py --repo Pliploop/remix-human-ratings
```

## API

| Route | Purpose |
| --- | --- |
| `GET /api/health` | liveness, and whether ratings are actually being persisted |
| `POST /api/ratings` | append one rating |
| `GET /api/progress/{annotator_id}` | assignments this rater has done (resume across browsers) |
